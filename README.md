# Crime-Pattern-Analysis-SQL-
**📌 Overview**

This project focuses on analyzing crime data using SQL to identify patterns, trends, and insights that can help in decision-making for law enforcement and public safety. The analysis includes crime frequency, location-based trends, time-based patterns, and category-wise distribution of crimes.

**🎯 Objectives**
Identify high-crime areas
Analyze crime trends over time
Find the most common crime types
Understand peak crime hours
Support data-driven decision-making

**🛠️ Tech Stack**
SQL (MySQL / PostgreSQL / SQL Server)
DBMS
Excel / CSV Dataset

**📂 Dataset**

The dataset includes the following fields:

Crime_ID
Date
Time
Location
Crime_Type
Description
Status (Solved / Unsolved)

**📊 Insights**
Certain locations show consistently higher crime rates
Crimes peak during specific hours of the day
Some crime types are more frequent than others
Monthly trends indicate seasonal variations

**📌 Conclusion**

This project demonstrates how SQL can be used to extract meaningful insights from crime data and support better decision-making.
